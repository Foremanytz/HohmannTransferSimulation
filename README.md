# 霍曼轉移軌道教學動畫（Hohmann Transfer）

一部互動式 3D 網頁動畫，呈現一艘太空船在兩顆行星之間「一往一返」的完整軌道轉移過程，用來講解 **霍曼轉移（Hohmann transfer）** 這個太空航行中最基本、最省燃料的變軌方法。整個動畫只需一個 HTML 檔，用瀏覽器打開即可播放。

---

## 這是什麼？

當一艘太空船想從一條繞行恆星的軌道，搬到另一條更高（或更低）的軌道時，並不是把船頭轉向目標、直線飛過去那麼簡單——在重力場裡，最省燃料的做法是只在兩個特定時刻各點一次火，讓船沿著一條剛好同時相切於出發軌道與目標軌道的橢圓「滑」過去。這條橢圓就叫 **霍曼轉移軌道**，是 1925 年由德國工程師 Walter Hohmann 提出的。

這部動畫把這個抽象的概念變成看得見的故事：你會看到太空船從第一顆行星出發、脫離它的重力、轉換軌道面、沿霍曼橢圓飛向第二顆行星、被第二顆行星捕獲，停留一段時間後再循同樣的邏輯飛回來。每一步都有繁體中文字幕說明「現在在做什麼、為什麼這樣做、目標是什麼、要花多少速度增量（ΔV）」。

不需要任何天文或物理背景也能看懂；想深入的人則可以從字幕與數據看到背後真實計算的軌道力學。

## 如何開啟

**最簡單的方式**：直接用瀏覽器（Chrome、Edge、Firefox、Safari 皆可）打開 `hohmann-transfer.html`，動畫會自動開始播放。

> 需要連網。動畫使用的 3D 繪圖函式庫（three.js）是從公開 CDN 載入的，因此第一次開啟時電腦需要能連上網際網路。

支援桌機與手機。手機上可用手指拖曳旋轉、雙指縮放。

## 動畫的情境設定

動畫裡有一顆恆星（位於中央）與兩顆行星，各自沿橢圓軌道繞著恆星公轉。以下是這個虛構星系的設定，動畫中所有軌跡都依這些數字精確計算：

| 項目 | 設定值 |
|------|--------|
| 行星一軌道 | 半短軸 1 AU、半長軸 1.5 AU（離心率 e ≈ 0.745，相當扁長） |
| 行星二軌道 | 半短軸 2 AU、半長軸 2.5 AU（離心率 e = 0.6） |
| 兩條軌道平面的夾角 | 20° |
| 兩軌道近日點方向的夾角 | 37° |
| 太空船推力上限 | 推重比 TWR = 2（以地球海平面重力為基準） |
| 燃料假設 | 不計損耗、不影響太空船質量（理想化，方便教學） |

> AU（天文單位）約等於地球到太陽的平均距離。「近日點(PE)」是軌道上離恆星最近的點，「遠日點(AP)」則是最遠的點。

因為這兩條軌道既不在同一平面上、又都是相當扁的橢圓，並不符合教科書裡「兩個共平面圓軌道」的理想霍曼前提。動畫採取的做法是先把太空船的軌道整理成接近教科書的條件，再執行霍曼轉移——這個處理過程本身也是動畫想傳達的重點之一。

## 完整任務流程

太空船的整趟旅程由 **八段機動** 組成：去程四段、回程四段（回程是去程的鏡像）。

**去程：行星一 → 行星二**

0. **脫離與圓化**：太空船在行星一的遠日點附近、選定的正確時刻點火，脫離行星一的重力，並把自己對恆星的軌道圓化在約 2.618 AU 的半徑上（即行星一遠日點所在的距離）。
1. **平面變換**：飛到兩條軌道平面的交線（節線）上時點火，把繞行方向從行星一的軌道面，轉到行星二的軌道面（轉過 20°）。
2. **霍曼注入**：點火進入霍曼轉移橢圓，朝行星二飛去，沿橢圓滑行半圈。
3. **捕獲**：抵達行星二時點火減速，進入繞行行星二的衛星軌道。

**停留**：在行星二的停泊軌道上等待回程的發射窗口。

**回程：行星二 → 行星一（機動 4–7）**：循與去程相同的邏輯鏡像操作——脫離行星二、平面變換、霍曼注入回內側、捕獲回行星一。

動畫會在出發與抵達時自動把鏡頭拉近到行星附近，讓你看清楚太空船繞行星的衛星軌道與脫離／捕獲的點火；轉移飛行時則把鏡頭拉遠到俯瞰整個星系的視角。

整趟任務的速度增量總和約為 **ΔV ≈ 49.2 km/s**，任務總時長約 **41.7 年**——其中回程光是等待正確的發射窗口就佔了約 30 年。動畫下方有一個年曆，每過一年會翻頁跳號，在這段漫長的等待中會快速翻頁，讓「太空旅行有多花時間」一目了然。

## 操作介面

- **播放／暫停、時間倍率**：控制動畫進行與快慢。動畫預設讓每一段機動約佔 10 秒，點火時會自動放慢並顯示火焰，同時標出「依 TWR = 2 換算，這段真實燃燒約 X 分鐘」。
- **視角切換**：俯視、斜視、跟隨太空船三種鏡頭。
- **任務序列軌**：畫面側邊列出八段機動的時間軸，點任一段即可跳到該機動。
- **字幕開關（🗨）**：可隨時隱藏或顯示底部的字幕框，方便在手機等小螢幕上把畫面完全讓給 3D 場景。
- **自由視角**：用滑鼠拖曳旋轉、滾輪縮放（手機為手指拖曳、雙指縮放）。

## 畫面元素說明

- **中央發光球**：恆星。
- **兩條軌道線與半透明平面**：兩顆行星各自的公轉軌道，以及它們所在的軌道平面（半透明的盤面讓你看出 20° 的夾角）。
- **節線標記**：兩個軌道平面的交線，平面變換就在這條線上發生。
- **行星與衛星軌道**：兩顆行星，以及太空船繞行星飛行的近距離軌道。
- **太空船與尾跡**：太空船本體與它走過的軌跡。
- **交會標記（綠色）**：霍曼轉移要求出發與抵達的「相位對齊」，這個標記指出兩者必須同時抵達的會合點。

## 技術說明

- **單一檔案**：所有 HTML、CSS、JavaScript 都包在 `index.html` 一個檔裡，沒有任何建置（build）步驟，也不需要安裝任何東西。
- **3D 繪圖**：使用 three.js（r128，自公開 CDN 載入）。
- **相機控制**：自製的拖曳旋轉／滾輪縮放／雙指縮放，不依賴額外外掛。
- **軌道計算**：兩顆行星的軌道以開普勒軌道參數描述；平面交線、發射窗口與各段速度增量都是以數值方法即時求解，而非預先寫死的動畫。

## 物理模型的簡化與限制

這部動畫的目標是「最容易理解」，而不是「最精密」或「最省燃料」。在使用或解說時請留意以下幾點刻意的簡化：

- 動畫的旁白在任務結束時會明白提醒：**這套機動程序是以最容易理解的方式安排的，並非以最低燃料消耗設計**——它鼓勵觀眾去思考有沒有更省的做法，這正是教學的用意。
- 點火採「脈衝近似」：把真實上需要數分鐘的燃燒，視為瞬間完成的速度改變（但軌道銜接是精確計算的），畫面上再把燃燒弧段的時長放大，否則在動輒數十年的尺度下根本看不見。
- 行星重力圈內（脫離與捕獲）的速度增量未計入帳面上的 ΔV 總和，這是常見的「拼接圓錐曲線（patched-conic）」簡化。
- 行星實際大小相對於整個星系極小，動畫中行星的半徑與衛星軌道都經過放大，純為示意。
